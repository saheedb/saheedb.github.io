Question 1

In your own words, compare and contrast a tibble with a dataframe (definition only)

Dataframes are used for storing data in table form with n cols and n rows. Tibbles are dataframes with some additional functionality and behave differently when subsetting and printing. Notably, printing tibbles allow viewing the data type in each column of the table.

2. Compare and contrast the following operations on a data.frame and equivalent tibble. What is different? Why might the default data frame behaviours cause you frustration?

df <- data.frame(abc = 1, xyz = "a")
df$x
[1] a
Levels: a
df[, "xyz"]
[1] a
Levels: a
df[, c("abc", "xyz")]

df$x may cause confusion because it returns the contents of df$xyz even though there is no x column. The contents of df$xyz is also coerced to a factor which may not be desirable and can be avoided by specifying stringsAsFactors = FALSE

df <- data.frame(abc = 1, xyz = "a", stringsAsFactors = FALSE)
df[, "xyz"]
[1] "a"

  1. What does tibble::enframe() do? When might you use it?

tibble::enframe() converts a vector or list to a dataframe. This may be used to organize the list output from functions such as lapply()

# enframe(unlist(lapply(mtcars, mean)), 
#         name  = "var", 
#         value = "mean")

lapply(mtcars, mean) %>%
  unlist() %>%
  enframe(name  = "var",
    value = "mean")
  1. What option controls how many additional column names are printed at the footer of a tibble?

Additional column names printed at the footer of a tibble can be specified using options(tibble.width = n, tibble.max_extra_cols = n)
